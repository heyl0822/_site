Structure of a program

Statement: instruction of a program.

Declaration statement: int x;

Assignment statement: x = 5;

Output statement: std::cout << x;

Expression: a mathematical entity that evaluates to a value.

Function: a collection of statements that executes sequentially.

hint: It’s a good idea to have your main() function live in a .cpp file with the same name as your project.

Library: a collection of precompiled code (e.g. functions) that has been “packaged up” for reuse in many different programs.

Syntax: rules that govern how sentences are constructed in a language

Syntax error: If you violate a rule, the compiler will complain

Comments

Comment: a line (or multiple lines) of text that are inserted into the source code to explain what the code is doing.

  • //: single-line comment, which tells the compiler to ignore everything to the end of the line.

  • /* */ multi-line comment. Everything in between the symbols is ignored. Do not nest.

  • At the library, program, or function level, describe what

  • Inside the library, program, or function, describe how

  • At the statement level, describe why.

Variables, initialization, and assignment

Variable: a name for a piece of memory that can be used to store information.

l-value: a value that has an address (in memory).

r-value: refers to any value that can be assigned to an l-value.

initialization: int x = 5;

assignment: int x; x = 5;

uninitialized variable: A variable that has not been assigned a value

Rule: Initialize your variables.

cout, cin, endl, the std namespace, and using statements

std::cout output text to the console

std::endl inserts a newline character

std::cin reads input from the user at the console using the input operator (»).

  • std::cin and cout always go on the left-hand side of the statement.

  • std::cout is used to output a value (cout = output)

  • std::cin is used to get an input value (cin = input)

  • « is used with std::cout, and shows the direction that data is moving from the r-value to the console.

  • >> is used with std::cin, and shows the direction that data is moving from the console into the variable.

using declaration

using directive

Rule: Avoid “using” statement outside of a function body.

Functions and return values

function: a reusable sequence of statements designed to do a particular job.

caller: The function initiating the function call

callee or called: the function being called

void: means the function does not return a value

status code: return a value back to the operating system. a return value of 0 means success, and a positive return value means failure.

Functions can not be defined inside other functions (called nesting) in C++.

function parameter: a variable used in a function where the value is provided by the caller of the function.

argument: a value that is passed from the caller to the function when a function call is made.

pass by value value of each of the arguments is copied into the matching parameter.

Benefits of functions

  • Organization
  • Reusability, avoid duplicate
  • Testing
  • Extensibility
  • Abstraction

Effectively using functions

  • Code that appears more than once in a program should generally be made into a function. For example, if we’re reading input from the user multiple times in the same way, that’s a great candidate for a function. If we output something in the same way multiple times, that’s also a great candidate for a function.

  • Code that has a discrete set of inputs and outputs is a good candidate for a function, particularly if it is complicated. For example, if we have a list of items that we want to sort, the code to do the sorting would make a great function, even if it’s only done once. The input is the unsorted list, and the output is the sorted list.

  • A function should generally perform one (and only one) task.

  • When a function becomes too long, too complicated, or hard to understand, it should be split into multiple sub-functions. This is called refactoring.

Operators

literal a fixed value that has been inserted (hardcoded) directly into the source code.

Operands supply the data that the expression works with. Literals, variables, and functions are all known as operands.

Operators tell the expression how to combine one or more operands to produce a new result.

Unary operators act on one operand. -5

Binary operators act on two operands (known as left and right). 3 + 4

Ternary operators act on three operands. (expression 1) ? expression 2 : expression 3

Whitespace and basic formatting

  • Your tabs should be set to 4 spaces

  • The braces that tell where a function begins and ends should be aligned with the function name, and be on their own lines

  • Each statement within braces should start one tab in from the opening brace of the function it belongs to.

  • Lines should not be too long. Typically, 72, 78, or 80 characters is the maximum length a line should be.

  • If a long line that is broken into pieces is broken with an operator (eg. « or +), the operator should be placed at the end of the line, not the start of the next line.

  • Use whitespace to make your code easier to read.

Forward declarations and definitions

Rule: When addressing compile errors in your programs, always resolve the first error produced first.

forward declaration allows us to tell the compiler about the existence of an identifier before actually defining the identifier.

function prototype consists of the function’s return type, name, parameters, but no function body

definition implements or instantiates (causes memory to be allocated for) the identifier.

declaration a statement that defines an identifier (variable or function name) and its type.

pure declarations a small subset of declarations that are not definitions, such as function prototypes

Header files

header guard prevent a given header file from being #included more than once from the same file.

.

< > angled brackets are used to tell the compiler that we are including a header file that was included with the compiler, so it should look for that header file in the system directories.

” “ The double-quotes tell the compiler that this is a header file we are supplying, so it should look for that header file in the current directory containing our source code first. If it doesn’t find the header file there, it will check any other include paths that you’ve specified as part of your compiler/IDE settings. That failing, it will fall back to checking the system directories.

Rule: Use angled brackets to include header files that come with the compiler. Use double quotes to include any other header files.

Rule: use the non .h version of a library if it exists, and access the functionality through the std namespace. If the non .h version does not exist, or you are creating your own headers, use the .h version. tell your compiler or IDE that you have a bunch of header files in some other location, so that it will look there when it can’t find them in the current directory. This can generally be done by setting an “include path” or “search directory” in your IDE project settings.

g++ -o main -I /source/includes main.cpp

  • Always include header guards.

  • Do not define variables in header files unless they are constants. Header files should generally only be used for declarations.

  • Do not define functions in header files.

  • Each header file should have a specific job, and be as independent as possible.

  • Give your header files the same name as the source files they’re associated with (e.g. grades.h goes with grades.cpp).

  • Try to minimize the number of other header files you #include in your header files. Only #include what is necessary.

  • Do not #include .cpp files.

preprocessor

Preprocessor a separate program that runs just before the compiler when you compile your program. Its purpose is to process directives. Directives are specific instructions that start with a # symbol and end with a newline

Includes #include <filename> #include "filename"

Macro defines a rule that defines how an input sequence.

  • Object-like macro:

#define identifier

#define identifier substitution_text

  • Function-like macros act like functions, and serve a similar purpose. We will not discuss them, because their use is generally considered dangerous, and almost anything they can do can be done by an (inline) function.

Conditional compilation #ifdef, #ifndef, and #endif.

#pragma once serves the same purpose as header guards, and has the added benefit of being shorter and less error-prone.

design programs

  • Step 1: Define the problem

  • Step 2: Define your tools, targets, and backup plan

    Understanding who your target users are and what they want.

    Defining what target architecture and/or OS your program will run on.

    Determining what set of tools you will be using.

    Determining whether you will write your program alone or as part of a team.

    Collecting requirements (a documented list of what the program needs to do).

    Defining your testing/feedback/release strategy.

    Determining how you will back up your code.

  • Step 3: Break hard problems down into easy problems

  • Step 4: Figure out the sequence of events

  • Step 5: Figure out the data inputs and outputs for each task

  • Step 6: Write the task details

  • Step 7: Connect the data inputs and outputs

  • Words of advice when writing programs

    Keep your programs simple to start.

    Add features over time.

    Focus on one area at a time.

    Test each piece of code as you go.

Debugging program

syntax error occurs when you write a statement that is not valid according to the grammar of the C++ language.

semantic error occurs when a statement is syntactically valid, but does not do what the programmer intended.

debugger a computer program that allows the programmer to control how a program executes and watch what happens as it runs.

Stepping a debugger feature that lets you execute (step through) your code line by line.

Step into executes the next line of code.

Step over executes the next line of code.

Step out executes all remaining code in the function you are currently in, and returns control to you when the function has finished executing.

Run to cursor executes the program like normal until it gets to the line of code selected by your cursor.

Run tell the debugger to run until it hits the end of the program

breakpoint a special marker that tells the debugger to stop execution of the program at the breakpoint when running in debug mode.

Watching variables the process of inspecting the value of a variable while the program is executing in debug mode.

call stack a list of all the active functions that have been called to get to the current point of execution.
